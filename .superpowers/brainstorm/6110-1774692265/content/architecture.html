Setup Claude Config — Architecture

How the slash command works end-to-end

Command Flow
1. User runs /setup-claude-config
→
Command prompt loaded
2. Scan base path
→
List projects as numbered menu
3. User picks project
→
Auto-detect tech stack
4. Present config preview
→
CLAUDE.md + settings.json + optional commands
5. User confirms/edits
→
Files written to project

Files Generated

CLAUDE.md
  • Project description
  • Tech stack
  • Coding conventions
  • Build/test/run cmds
  • File structure
  • Permissions & rules
.claude/settings.local.json
  • Stack-based permissions
  • Bash allow rules
  • Model preferences
.claude/commands/ (opt-in)
  • run-tests.md
  • build.md
  • lint.md
  • Stack-specific cmds
